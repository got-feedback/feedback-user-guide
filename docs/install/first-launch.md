# First Launch

Short answer: the first launch may take longer than normal while FeedBack prepares local resources, plugins, and model caches.

## What To Expect

- The app may take a minute to open the first time.
- Some audio or machine-learning features may initialize lazily.
- You may be taken directly into first-run setup.
- Later launches should be faster.

## First-Run Checklist

1. Confirm the app opens.
2. Complete audio input setup.
3. Set your instrument and tuning if prompted.
4. Import or scan at least one song.
5. Open the tuner and confirm input signal.
6. Play a short song or section.

## Common Problems

| Problem | Try This |
|---|---|
| App seems slow on first launch | Wait a minute before force-closing; first-run setup can take longer. |
| App never opens | See [Desktop Install Problems](../troubleshooting/desktop-install.md). |
| First-run setup cannot hear input | See [No Signal](../troubleshooting/no-signal.md). |
