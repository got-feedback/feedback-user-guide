# FeedBack 0.3.0 Notes

This page is the user-facing launch page for FeedBack 0.3.0. Keep it short and link to task pages for instructions.

## Highlights To Confirm Before Publishing

- First-run audio setup improvements.
- Clearer audio input labels on supported desktop builds.
- Library refresh/rescan behavior.
- Playlist and queue behavior.
- Tuner behavior and auto-open settings.
- Metadata/library organization changes that are included in the final release.
- Known limitations and tester caveats.

## Recommended Launch Copy Shape

### Better First-Run Setup

FeedBack 0.3.0 focuses on getting players into a working song with less guesswork. The user guide should point new users through install, audio input selection, tuning, calibration, song import, and first playback.

### Library Improvements

The library docs should explain scan vs rescan, importing song files, playlists, queue behavior, and any album/metadata work that ships in the final build.

### Tuning and Input

The tuner and audio setup pages should explain current-song tuning, manual tuner use, input device labels, and what to do when FeedBack hears the wrong device.

### Troubleshooting and Diagnostics

Make diagnostics easy to find. Tester reports should include build, platform, input device, steps, screenshots, and the diagnostics bundle.
