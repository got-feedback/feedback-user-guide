# Style Guide

## Voice

- Be direct and practical.
- Tell users what to click and what should happen.
- Prefer short sections and tables for troubleshooting.
- Avoid implementation details unless the page is for advanced users.

## Terms

- Use "FeedBack" for the app.
- Use "feedpak" for the song package format.
- Use "song file" before "feedpak" on beginner pages.
- Use "arrangement" for the playable part inside a song.
- Use "plugin" for add-on tools/views.

## Screenshots

- Add placeholders first.
- Capture final screenshots after the UI is stable.
- Include platform notes when UI differs by platform.

## Page Status

Use simple status language in comments or front matter if the site generator supports it:

- Draft
- Stub
- Ready for screenshot
- Ready for review
- Published
